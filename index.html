-- server-authoritative state transition
function CombatState.transition(player, toState)
    local current = StateMap[player.UserId]
    if not isValidTransition(current, toState) then
        return false, "illegal transition"
    end

    StateMap[player.UserId] = toState
    Signal:Fire("StateChanged", player, toState)
    return true
end

CombatSystem.luau
[ Combat · FSM · Hit Detection ]
RemoteEventFSMServer-authHitbox

Server-authoritative melee combat built on a five-state finite-state machine. All state transitions are validated server-side — clients request, the server decides. Hitboxes are owned and resolved by the server only, which eliminates the entire class of client-sided hit-registration exploits.

Designed around explicit guard conditions so illegal transitions (e.g. attacking mid-stun) are rejected with a reason string rather than silently falling through.

InventorySystem.luau

              
[ Inventory · DataStore · Session Locking ]

              

                ProfileServiceDataStoreAttributes
              

            

            

              
Session-locked inventory framework built on ProfileService. Profiles are locked to a single server session to prevent duplicate writes during server overlap, and reconciled against a template on load so missing keys never cause runtime errors.

              
Items are pure data tables in a central registry — no hardcoded if/elseif chains. Any system that needs item stats reads the attribute table directly, keeping the inventory module decoupled from combat and crafting logic.

MatchmakingSystem.luau

              
[ Matchmaking · MemoryStore · Teleport ]

              

                MemoryStoreTeleportServiceCross-server
              

            

            

              
Skill-based matchmaking across Roblox servers using MemoryStore's SortedMap as a shared queue. All queue mutations use atomic UpdateAsync calls to prevent race conditions between concurrent servers.

              
A lightweight per-tick lock ensures only one server runs the matchmaking pass at a time. Skill tolerance widens the longer a player waits, so queues never stall in low-population regions — with a configurable cap to prevent wildly mismatched games.
